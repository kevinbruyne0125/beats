BEATS

is a drum machine written in pure Ruby. Feed it a song notated in YAML, and it will generate a Wave file possessing impeccable timing and feel. Here is an example song:

Song:
  Tempo: 120
  Structure:
    - Verse:   x2
    - Chorus:  x2
    - Verse:   x2
    - Chorus:  x4

Verse:
  bass.wav:    X...X...X...XX..X...X...XX..X...
  snare.wav:   ..X...X...X...X.X...X...X...X...
  hihat.wav:   X.X.X.X.X.X.X.X.X.X.X.X.X.X.X.X.
  cymbal.wav:  X...............X..............X

Chorus:
  bass.wav:    X...X...XXXXXXXXX...X...X...X...
  snare.wav:   ...................X...X...X...X
  hihat.wav:   X.X.XXX.X.X.XXX.X.X.XXX.X.X.XXX.
  cymbal.wav:  ........X.......X.......X......X
  sine.wav:    ....X...................X......X

Installation and Usage

Here are instructions for installation and usage.

Creating Songs

Songs are notated using YAML, so each song file is just a plain text file.

Each song is divided into one or more patterns. A pattern represents a section of the song, such as a verse, chorus, or bridge. Each pattern in turn has one or more tracks. Each track represents an individual sound, such as a bass drum, snare, hi-hat, etc.

Songs should start with a header indicating the tempo and the order that the patterns should be played. Patterns can optionally be repeated one or more times. For example, adding "x2" after the pattern name means to play it twice. Since this is YAML, lines must be indented correctly. Also, each pattern in the structure should be preceded by a dash. Here is an example header:

Song:
  Tempo: 120
  Structure:
    - Verse
    - Chorus: x2
    - Verse:  x3
    - Chorus: x4
    - Verse:  x2

One or more patterns should follow the header. (If there are no patterns, there is nothing to play!) Each pattern definition starts with the pattern name, followed by the specification for each track on separate lines.

Verse:
  bass.wav:    X...X...X...XX..X...X...XX..X...
  snare.wav:   ..X...X...X...X.X...X...X...X...
  hihat.wav:   X.X.X.X.X.X.X.X.X.X.X.X.X.X.X.X.
  cymbal.wav:  X...............X..............X

Each track line starts with the sound file to use (only Wave files are supported), followed by the rhythm. Once again, lines must be indented properly. You can use any Wave file you have on your computer as a sound. If the Wave file is not in the current working directory, be sure to include the full path.

In the rhythm, an X indicates that the sound should be triggered, and a . indicates a "rest" between triggers. Each X or . represents a 16th note. You only need to indicate the downbeat of each sound, since sounds will play their full duration (unless cut off by another trigger). For example, to play a quarter note rhythm, X...X...X...X... will do the trick.

As a final note, patterns and the song header can come in any order. If you want to include the song header at the end of the file, you can. You can also mix up the order that patterns are defined.
